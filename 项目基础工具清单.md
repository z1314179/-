# 项目基础工具清单

本文档整理当前项目中常用的基础能力模块，便于快速定位与复用。

## 多标签与页面缓存（重要约定）

- **系统标签栏**：布局顶部的多标签对应 `src/components/Layout/index.vue` 中的标签列表与 `keep-alive`。菜单打开后若出现**系统标签**，说明该路由已纳入标签体系；在路由配置了 **`meta.keepAlive`**（并由 `src/store/tabs.js` 维护 `includeList`）时，页面会被 **缓存**，组件实例会被复用。
- **可缓存页面的表现**：再次从标签切回该页时，**不会再次执行 `onMounted`**，仅会触发 **`onActivated`**（若组件被 `keep-alive` 包裹）。
- **希望每次切回都刷新**：在页面中使用 Vue 的 **`onActivated`** 生命周期里调用列表刷新（例如 `getList()`）、或按需重置局部状态；不要只依赖 `onMounted` 拉数，否则切回标签时数据不会更新。

## 1. 网络请求层

- `src/http/index.js`

  - 请求拦截器自动注入 `Authorization: Bearer <token>`

## 2. 状态管理（Pinia）

- `src/store/user.js`

- `src/store/index.js`

## 8. 使用建议

- **多标签 / 缓存**：见上文「多标签与页面缓存」；列表类页面切回标签时若需刷新，在 **`onActivated`** 中调用 `getList()` 等与 `useTable` 配套的方法。
- 列表页优先复用 `useTable.js` + `SearchCard.vue`
- 权限控制优先复用 `usePerm.js` 与 `directives.js`
- 文件导入导出优先使用 `leadInFile.js` / `exportExcle.js` / `preview.js`
- 新接口统一放 `src/api/*.js`，由 `src/http/index.js` 统一拦截处理
- 后续业务需求如可复用，优先使用本文档中的公共配置、公共组件与工具能力，避免重复开发
